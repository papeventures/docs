---
title: "REST API endpoints for ads, agencies, and users"
description: "Retrieve individual ad creative records, search and filter agency records, and list or look up MediaOS user accounts via the REST API."
---

This page covers three supporting resource endpoints: **ads**, **agencies**, and **users**. These are companion resources to contracts, contacts, and advertisers — you will typically use IDs returned by those endpoints to look up records here.

All list requests support pagination and sorting. See [Pagination and Sorting](/api/pagination) for details.

---

## Ads

An ad record represents a creative file associated with an ad placement. The ads endpoint is read-only and supports retrieval by ID only — there is no list endpoint.

### Get a single ad

```
GET /ads/{id}
```

Returns a single ad record by its numeric ID.

#### Path parameters

<ParamField path="id" type="integer" required>
  The numeric ID of the ad.
</ParamField>

#### Example request

```bash
curl -u "username:password" \
  https://yourcompany.mediaosnow.com/API/ads/9901
```

#### Example response

```json
{
  "id": 9901,
  "name": "Full Page Spring Campaign",
  "text": null,
  "advertiserID": 1042,
  "url": "https://files.mediaosnow.com/tmp/acme-full-page-spring.pdf?token=abc123",
  "createDate": "2024-01-10",
  "deleteDate": null,
  "preFlighted": true
}
```

#### Response fields

<ResponseField name="id" type="integer">
  Unique identifier for the ad.
</ResponseField>

<ResponseField name="name" type="string">
  Display name for the ad creative.
</ResponseField>

<ResponseField name="text" type="string">
  Optional text content associated with the ad, if any.
</ResponseField>

<ResponseField name="advertiserID" type="integer">
  The ID of the advertiser who owns this creative.
</ResponseField>

<ResponseField name="url" type="string">
  A temporary download link to the production file. This URL expires — do not cache it.
</ResponseField>

<ResponseField name="createDate" type="string">
  Date the ad record was created.
</ResponseField>

<ResponseField name="deleteDate" type="string">
  Date the ad was deleted, or `null` if it is still active.
</ResponseField>

<ResponseField name="preFlighted" type="boolean">
  Whether the ad creative has been pre-flighted (reviewed for print/digital specifications).
</ResponseField>

## Agencies

An agency is an advertising agency that may be associated with contacts and contracts. Agency endpoints support listing, filtering, and individual lookup.

### List agencies

```
GET /agencies/
```

Returns a paginated list of agency records matching the supplied filters.

#### Query parameters

<ParamField query="id" type="integer">
  Return the agency with this exact ID.
</ParamField>

<ParamField query="name" type="string">
  Filter by agency name using wildcard pattern matching (for example, `Acme%`).
</ParamField>

#### Sorting

Valid `sort_by` values for this endpoint:

| Value | Description |
|---|---|
| `id` | Sort by agency ID (default) |
| `name` | Sort by agency name |
| `date_created` | Sort by creation date |

Default sort order is `desc`.

#### Example request

```bash
curl -u "username:password" \
  "https://yourcompany.mediaosnow.com/API/agencies/?sort_by=name&sort_order=asc"
```

#### Example response

```json
{
  "agencies": [
    {
      "id": 15,
      "name": "Springfield Ad Group",
      "date_created": "2021-08-10",
      "discount_percent": 15,
      "primary_address": {
        "id": 60,
        "address1": "456 Oak Ave",
        "city": "Springfield",
        "state": "IL",
        "zip": "62702"
      },
      "primary_phone": { "id": 140, "value": "555-867-5309" },
      "primary_email": { "id": 141, "value": "hello@springfieldadgroup.example.com" }
    }
  ],
  "count": 1
}
```

#### Response fields

<ResponseField name="id" type="integer">
  Unique identifier for the agency.
</ResponseField>

<ResponseField name="name" type="string">
  The agency's display name.
</ResponseField>

<ResponseField name="date_created" type="string">
  Date the agency record was created.
</ResponseField>

<ResponseField name="discount_percent" type="number">
  The standard agency commission discount percentage.
</ResponseField>

<ResponseField name="primary_address" type="object">
  The agency's primary mailing address.
</ResponseField>

<ResponseField name="primary_phone" type="object">
  The agency's primary phone contact data record.
</ResponseField>

<ResponseField name="primary_email" type="object">
  The agency's primary email contact data record.
</ResponseField>

### Get a single agency

```
GET /agencies/{id}
```

#### Path parameters

<ParamField path="id" type="integer" required>
  The numeric ID of the agency.
</ParamField>

#### Example request

```bash
curl -u "username:password" \
  https://yourcompany.mediaosnow.com/API/agencies/15
```

## Users

The users endpoints let you list and look up MediaOS user accounts in your organization. Users are referenced throughout the API as owners, creators, and sales representatives.

### List users

```
GET /users/
```

Returns a paginated list of user records matching the supplied filters.

#### Query parameters

<ParamField query="id" type="integer">
  Return the user with this exact ID.
</ParamField>

<ParamField query="first_name" type="string">
  Filter by first name using wildcard pattern matching (for example, `Acme%`).
</ParamField>

<ParamField query="last_name" type="string">
  Filter by last name using wildcard pattern matching (for example, `Acme%`).
</ParamField>

<ParamField query="email" type="string">
  Filter by email address using wildcard pattern matching (for example, `Acme%`).
</ParamField>

<ParamField query="active" type="boolean">
  Filter by active status.
</ParamField>

<ParamField query="is_administrator" type="boolean">
  When `true`, return only users with the Administrator role.
</ParamField>

<ParamField query="is_sales" type="boolean">
  When `true`, return only users with the Sales role.
</ParamField>

<ParamField query="is_accounting" type="boolean">
  When `true`, return only users with the Accounting role.
</ParamField>

<ParamField query="is_production" type="boolean">
  When `true`, return only users with the Production role.
</ParamField>

#### Sorting

Valid `sort_by` values for this endpoint:

| Value | Description |
|---|---|
| `id` | Sort by user ID (default) |
| `firstName` | Sort by first name |
| `lastName` | Sort by last name |
| `email` | Sort by email address |

Default sort order is `desc`.

#### Example request

```bash
curl -u "username:password" \
  "https://yourcompany.mediaosnow.com/API/users/?is_sales=true&active=true&sort_by=lastName&sort_order=asc"
```

#### Example response

```json
{
  "users": [
    {
      "id": 7,
      "first_name": "Bob",
      "last_name": "Jones",
      "email": "bob@yourcompany.example.com",
      "is_admin": false,
      "is_accounting": false,
      "is_production": false,
      "is_metrics": false,
      "is_members": false,
      "is_sales_manager": false,
      "is_sales": true,
      "is_editorial": false,
      "active": true,
      "date_created": "2020-05-12",
      "date_deleted": null
    }
  ],
  "count": 1
}
```

#### Response fields

<ResponseField name="id" type="integer">
  Unique identifier for the user.
</ResponseField>

<ResponseField name="first_name" type="string">
  The user's first name.
</ResponseField>

<ResponseField name="last_name" type="string">
  The user's last name.
</ResponseField>

<ResponseField name="email" type="string">
  The user's login email address.
</ResponseField>

<ResponseField name="is_admin" type="boolean">
  Whether the user has the Administrator role.
</ResponseField>

<ResponseField name="is_accounting" type="boolean">
  Whether the user has the Accounting role.
</ResponseField>

<ResponseField name="is_production" type="boolean">
  Whether the user has the Production role.
</ResponseField>

<ResponseField name="is_metrics" type="boolean">
  Whether the user has access to Metrics.
</ResponseField>

<ResponseField name="is_members" type="boolean">
  Whether the user has access to Members.
</ResponseField>

<ResponseField name="is_sales_manager" type="boolean">
  Whether the user has the Sales Manager role.
</ResponseField>

<ResponseField name="is_sales" type="boolean">
  Whether the user has the Sales role.
</ResponseField>

<ResponseField name="is_editorial" type="boolean">
  Whether the user has the Editorial role.
</ResponseField>

<ResponseField name="active" type="boolean">
  Whether the user account is currently active.
</ResponseField>

<ResponseField name="date_created" type="string">
  Date the user account was created.
</ResponseField>

<ResponseField name="date_deleted" type="string">
  Date the user was deactivated, or `null` if the account is still active.
</ResponseField>

### Get a single user

```
GET /users/{id}
```

#### Path parameters

<ParamField path="id" type="integer" required>
  The numeric ID of the user.
</ParamField>

#### Example request

